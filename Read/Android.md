# 1. Binder

## 1.1 为什么Android要采用Binder

管道：复制两次 点对点模式

socket： 复制两次 C/S

共享内存： 效率高 不安全

|        | Binder                              | 共享内存                               | Socket                                 |
| ------ | ----------------------------------- | -------------------------------------- | -------------------------------------- |
| 性能   | 需要拷贝一次                        | 无需拷贝                               | 拷贝两次                               |
| 特点   | 基于C/S架构 易用性高                | 控制复杂 易用性差                      | 基于C/S架构 易用性差                   |
| 安全性 | 为每个APP分配uid 同时支持实名和匿名 | 依赖上层协议 访问接入点是开放的 不安全 | 依赖上层协议 访问接入点是开放的 不安全 |
